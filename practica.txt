1.	Вивчення теорії

Test Case: Документ, який описує кроки, умови і очікувані результати для перевірки певної функціональності.
Test Plan: План тестування, який включає цілі, стратегію, ресурси та графік тестування.
Bug Report: Документ для фіксації знайденої помилки в програмі (опис, кроки для відтворення, очікуваний і фактичний результат).
Check List: Список завдань чи перевірок для контролю тестового процесу.



2.	Завантажив додаток.
 

Помилки які я знайшов:
1. Дублювання функціоналу:
 Поруч з полем “Range” присутнє випадаюче меню, але його використання не очевидне. Воно не має жодних пояснень чи інструкцій, що робить його незрозумілим для користувача.
2. Відсутність пояснень для елементів управління:
Поле “Range”, опції сортування (“Sort Order”) та символи (“Symbols”) не мають пояснення їхньої функції.
Користувач не може зрозуміти, як використовувати ці функції без документації чи додаткових підказок.
3. Некоректний дизайн:
 Немає чіткого поділу між розділами (“Sort Order”, “Symbols” тощо). Це може створити плутанину для користувача.
Кнопка “Add to List” виглядає активною, а кнопка “Clear List” — неактивною, хоча не зрозуміло, чому. А також кнопка Edit з помилкою.
